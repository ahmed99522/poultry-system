# نظام إدارة طلبات وتوزيع المنتجات (PoultryFlow)

نظام ويب متكامل (PWA) لإدارة دورة العمل الكاملة: تسجيل الطلبات → إغلاقها → تسجيل الأوزان تلقائيًا مع توليد كود لكل طائر وحساب السعر → التوزيع على العملاء → إصدار الفواتير، مع صلاحيات مختلفة لكل نوع مستخدم (مدير، موزّع، موظف أوزان).

تم بناء المشروع بالكامل واختباره فعليًا (تسجيل دخول، إنشاء طلب، إغلاقه، تسجيل أوزان 3 طيور، توزيعها، إصدار فاتورة بمبلغ صحيح تلقائيًا) ويعمل بدون أي أخطاء.

---

## 1) البنية التقنية

| الطبقة | التقنية | السبب |
|---|---|---|
| Backend | Node.js + Express | خفيف، سريع، سهل النشر على أي استضافة |
| قاعدة البيانات | SQLite (better-sqlite3) | ملف واحد، بدون خادم منفصل، أداء عالٍ، نسخ احتياطي بنسخ ملف واحد. قابلة للترقية لاحقًا لـ PostgreSQL دون تغيير منطق العمل |
| المصادقة | JWT (access + refresh) | access token عمره 15 دقيقة، refresh token في httpOnly cookie آمن لمدة 7 أيام |
| Frontend | React + Vite + TailwindCSS | واجهة حديثة، سريعة البناء، تصميم احترافي قابل للتخصيص |
| PWA | vite-plugin-pwa (Workbox) | يعمل من أي جهاز بدون تثبيت من متجر تطبيقات، ويعمل جزئيًا بدون إنترنت |
| اللغة | react-i18next | تبديل فوري عربي ⇄ إنجليزي مع تبديل اتجاه الصفحة RTL ⇄ LTR |

## 2) الأمان (Security) — أهم ما تم تطبيقه

- **كلمات المرور**: مشفّرة بـ bcrypt (12 rounds)، لا تُخزَّن أبدًا نصًا صريحًا.
- **الجلسات**: access token قصير العمر (15 دقيقة) + refresh token في httpOnly + Secure + SameSite=strict cookie (لا يمكن لجافاسكريبت الوصول له، يحمي من XSS).
- **منع Brute Force**: قفل الحساب تلقائيًا بعد 5 محاولات فاشلة لمدة 15 دقيقة + Rate Limiting صارم على مسار تسجيل الدخول (10 محاولات/15 دقيقة لكل IP).
- **منع SQL Injection**: 100% من الاستعلامات تستخدم Prepared Statements (better-sqlite3) — لا يوجد أي دمج نصي مباشر للمدخلات في الاستعلامات.
- **التحقق من المدخلات**: كل مسار يستخدم مكتبة Zod للتحقق الصارم من شكل ونوع كل حقل قبل معالجته.
- **رؤوس HTTP الأمنية**: عبر Helmet (Content-Security-Policy, HSTS, X-Frame-Options...).
- **CORS محدد**: فقط النطاقات المسجّلة صراحة في `CORS_ORIGINS` يمكنها الاتصال بالـ API.
- **صلاحيات دقيقة (RBAC)**: كل مسار API يتحقق من دور المستخدم (`requireRole`) قبل التنفيذ، وليس فقط من تسجيل الدخول.
- **سجل تدقيق كامل (Audit Log)**: كل عملية حساسة (دخول، فشل دخول، إنشاء/تعديل/حذف، تغيير كلمة مرور، توزيع، فوترة) تُسجَّل مع المستخدم والوقت وعنوان IP.
- **رسائل خطأ غير كاشفة**: مثل رسالة "اسم المستخدم أو كلمة المرور غير صحيحة" الموحّدة لمنع اكتشاف أسماء المستخدمين (Username Enumeration).
- **حماية HPP**: منع هجمات تلوث معاملات HTTP.

### قبل الرفع على الإنترنت — تأكد من:
1. توليد مفاتيح JWT قوية وعشوائية بالكامل (`openssl rand -hex 64`) ووضعها في `.env` — **لا تستخدم القيم الافتراضية أبدًا في الإنتاج**.
2. تغيير كلمة مرور المدير الافتراضية فور أول تسجيل دخول.
3. تشغيل الموقع خلف HTTPS إجباري (شهادة SSL مجانية عبر Let's Encrypt).
4. ضبط `NODE_ENV=production` في الباك اند.
5. أخذ نسخة احتياطية دورية من ملف `backend/data/poultry.db`.

## 4) التشغيل محليًا (خطوة بخطوة)

### الباك اند
```bash
cd backend
npm install
cp .env.example .env
# افتح .env وضع مفاتيح JWT قوية:
# JWT_ACCESS_SECRET=<ناتج openssl rand -hex 64>
# JWT_REFRESH_SECRET=<ناتج آخر مختلف>
npm run seed     # ينشئ حساب المدير الافتراضي (admin / ChangeMe@2026)
npm run dev       # أو: npm start
```
السيرفر يعمل على: `http://localhost:4000`

### الفرونت اند
```bash
cd frontend
npm install
cp .env.example .env   # VITE_API_URL=http://localhost:4000/api
npm run dev
```
الموقع يعمل على: `http://localhost:5173`

سجّل الدخول بـ: **admin** / **ChangeMe@2026** ثم غيّر كلمة المرور فورًا من الإعدادات.

## 5) دورة العمل داخل النظام

1. **الموزّع** يفتح "الطلبات" وينشئ طلبًا جديدًا: يختار العميل + المنتج + الكمية المطلوبة.
2. عندما يكتمل استلام الكمية، يضغط الموزّع **"إغلاق الطلب"** → النظام يولّد تلقائيًا شريحة لكل طائر بكود فريد بانتظار الوزن.
3. **موظف الأوزان** يفتح شاشة "الأوزان" السريعة (خانة إدخال واحدة تركّز تلقائيًا) — يكتب الوزن ويضغط Enter، ينتقل تلقائيًا للطائر التالي. النظام يحسب السعر تلقائيًا (الوزن × سعر الكيلو).
4. **الموزّع** يفتح شاشة "التوزيع" ويرى كل الطيور الموزونة، ويسند كل طائر إما لعميل الطلب الأصلي أو لعميل آخر إذا كان وزنه أنسب له.
5. **الموزّع/المدير** يفتح "الفواتير" ويصدر فاتورة لكل عميل ضمن الطلب — يحسب النظام تلقائيًا عدد الطيور والوزن الكلي والمبلغ الإجمالي.
6. كل شيء مسجّل بالكامل في سجل التدقيق، وكل الشاشات تدعم العربية/الإنجليزية عبر أيقونة تبديل اللغة في الأعلى.

## 6) النشر على الإنترنت (خيارات مقترحة)

### خيار سريع ومجاني للتجربة
- **Backend**: Render.com / Railway.app (Web Service من مستودع Git) — يدعم SQLite عبر Persistent Disk.
- **Frontend**: Vercel / Netlify / Cloudflare Pages — ينفذ `npm run build` وينشر مجلد `dist`.
- اضبط `VITE_API_URL` في الفرونت اند على رابط الباك اند المنشور، واضبط `CORS_ORIGINS` في الباك اند على رابط الفرونت اند المنشور.

### خيار احترافي كامل التحكم (VPS)
1. سيرفر Ubuntu + Nginx كـ reverse proxy + Let's Encrypt (Certbot) لشهادة SSL مجانية.
2. تشغيل الباك اند بـ PM2 (`pm2 start server.js --name poultry-api`) ليعمل باستمرار ويعيد التشغيل تلقائيًا عند الأعطال.
3. بناء الفرونت اند (`npm run build`) وتقديم مجلد `dist` مباشرة عبر Nginx.
4. Nginx يوجّه `/api/*` إلى الباك اند (المنفذ 4000) ويقدّم باقي الملفات كـ static من `dist`.

## 7) الأدوار والصلاحيات

| الدور | الصلاحيات |
|---|---|
| **مدير (admin)** | كل شيء: المستخدمين، المنتجات، العملاء، الطلبات، الأوزان، التوزيع، الفواتير |
